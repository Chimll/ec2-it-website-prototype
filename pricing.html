Pricing guide

How much does IT support cost?

Pricing should reflect the support scope, systems, security expectations and management responsibilities involved. These figures are indicative and subject to discovery.

Why pricing varies

Two firms with the same user count can need very different levels of support.

Costs change when a business has multiple offices, legacy systems, server infrastructure, demanding security requirements, frequent projects, complex supplier arrangements or specific response expectations. A clear pricing discussion should show what is included, what is separate and how work is handled.

What changes the cost

  • User count and number of devices
  • Number of sites and remote working patterns
  • Microsoft 365, server and cloud complexity
  • Security tooling and backup requirements
  • Required review cadence and senior input
  • Out-of-hours or high-priority coverage needs

What is normally included

  • Helpdesk access within agreed support hours
  • Remote support and administration
  • Monitoring and patching oversight where scoped
  • Microsoft 365 admin support
  • Service review and practical advice

What is normally separate

  • Hardware, software and cloud licences
  • Major projects, migrations and office moves
  • Specialist consultancy outside support scope
  • Remediation of known legacy issues
  • Out-of-hours support unless agreed in advance

What to watch for when comparing MSP quotes

  • Whether scope is clear or hidden behind vague wording
  • Whether security tooling is included or optional
  • How projects and changes are charged
  • How onboarding, documentation and reviews are handled
  • Whether the supplier understands your working environment
